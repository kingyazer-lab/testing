# SoloDay Web MVP - P0级别重构版

## 项目概述
基于P0级别修改意见重构的SoloDay Web MVP应用，专注于减压、简化和降指令感。

## P0修改要点

### 1. 首页"减压"：状态从"可操作"改为"展示"
- ✅ 首页状态展示为只读模式，不再提供选择按钮
- ✅ 状态调整功能移至【我的】页面
- ✅ 减少用户决策压力

### 2. 登录流程简化：去掉"用户名+密码"
- ✅ 移除传统登录流程
- ✅ 实现"直接开始"模式
- ✅ 昵称输入为可选功能
- ✅ 使用localStorage保存用户数据

### 3. 首页只保留"一屏"，不堆信息
- ✅ 首页仅包含4个核心元素：
  - 标题「今日生活」
  - 状态展示（1行）
  - 三件事卡片（吃/清洁/放松）
  - 底部"允许感"文案
- ✅ 移除统计、进度条、完成率、连续天数等信息

### 4. 按钮/提示语"降指令感"（文案替换）
- ✅ 三张卡片文案统一替换为建议性语言：
  - 🍳 吃一顿简单的 - 副文案：15 分钟 · 一人份 - 按钮：看看今天吃什么
  - 🧽 清洁一点点（主推荐）- 副文案：10–15 分钟就够 - 按钮：我现在做
  - 🌿 放松一下 - 副文案：停 10 分钟就好 - 按钮：开始休息
- ✅ 完成后反馈文案采用"允许感"表达：
  - 「可以了，今天不用再管这个」
  - 「做到了就行，不必做到完美」
  - 「你已经把生活接住了」

## 技术实现

### 文件结构
- `index.html` - 主页面结构
- `styles.css` - 样式文件（极简MUJI+Apple风格）
- `data.js` - 数据结构和工具函数
- `app.js` - 应用逻辑和交互

### 核心功能
1. **简化登录**：直接进入应用，无需账号密码
2. **状态展示**：首页只展示当前状态，不进行操作
3. **三件事卡片**：吃、清洁、放松三个核心功能
4. **状态管理**：在【我的】页面调整状态
5. **数据持久化**：使用localStorage保存用户数据

### 设计风格
- 极简MUJI+Apple风格
- 大量留白，克制的设计
- 中性色调，清晰字体
- 响应式设计，支持手机访问

## 使用说明

1. 打开应用后点击"直接开始"即可使用
2. 可选择输入昵称（可选）
3. 在首页查看当前状态和三件事建议
4. 点击卡片进入对应功能页面
5. 在【我的】页面调整状态和设置

## 数据存储
所有用户数据通过localStorage保存在本地，包括：
- 用户状态（稳/累/乱/空）
- 昵称（可选）
- 设置偏好（清洁提醒、饮食建议）
- 清洁记录时间

## 注意事项
- 这是一个前端演示应用，无后端服务
- 所有数据存储在浏览器本地
- 清除浏览器数据会重置应用状态
- Pages trigger
